The main purpouse of this repository is to serve as a workspace to facilitate IBM hardware quantum krylov calculations
for molecular hamiltonains generated by qforte.

IMPORTANT. This is not meant to be a produciton package, and all scripts (qforte and qiskit) should above all else be simple, and human readable.
Please avoid defining dozens of auzillary funcitons, in-line solutions are always prefered as they are easier to read without
moving ones eyes to another part of the file (or god forbid another file alltogether)
Ideally we will never have a simple python file that is longer that a few hunderd lines.
Every script should also have comments at the top that give the exact command one would use to
manually re-run it along with a written summary of what it does and what the relevant user facing optoins are.
NO FLAGS SHOULD BE USED, ALWAYS HARD CODED USER OPTIONS. 

IMPORTANT. Much of the mathematical nuance and technical detail required to successfully achieve the aims of this repository
are NOT included in the present file. As such the qforte codebase (local on this machine) and stochastic_srqk.tex should be regularly 
consulted when creating prompts and generating the below scripts.

IMPORTANT. There are already conda environments on this machine that should be able to facilitate running of the below scripts.
Qforte-centric jobs should be run with the qfe_env_v1 anaconda environment (1-6 below)
Qiskit-centric jobs should be run with the qiskit_env_v1 anaconda environment (6-10 below)

Before we begin we will need to set up the scaffolding for an intelegent directory struction.

Workflow:

1. Generate a molecule object in qforte, linear H4 for now in sto-3g basis, using psi4 as a backend, save molecule metadata
to a json file (or similar) including Ehf, and Efci for perhaps 8 roots.

2. Build the "hermitian pairs" (HP) form of hamiltonain as a qforte.SQOpPool using qforte

H = h_o + sum_mu h_mu Hmu = h_o sum_mu h_mu (g_mu + g_mu^)

where g_mu is a fermionic monomial, and h_o is the zero body energy (usually nuclear repulsion energy)

3. Port each term of the qforte HP hamiltonian to an openfermion FermionOperator, and then use the Jordan Wigner
Mapping to translate heach h_mu H_mu -> sum_l h_mu_l P_mu_l

4. Save each term in a jason file (or eqivalent)

5. In a new script, read in the jason or similar file from 4. and create transpiled qiskit circuits for
each U_mu = e^-i dt h_mu H_mu. There should be an easily toggleable user keyword that can be used to specify
what hardware target the transpilation pertains to.

6. Save the transpiled circuits to a file (appropriate format)

==> Read in transpiled circuits, build quantum krylov matrix elements, save matricies <====

7. In another new script read in the transpiled circuits such that they can optionally be executed A) on a qiskit simulator
backend (with or without noise/error modeling, more user keywords), and B) on an actyal quantum processing unit.
This script importantly will also implement the protocols to estimate the matrix elements of the unitary quantum
krylov diagonalization using sampling only (specifically, using the multifidility estimation protocal outlined in
stochastic_srqk.tex). For now the deliverable can be production of the "overlap" matrix S, and we can assume toeplitz
struction such that we ony actyally need to determine expectaiotns of subsequently longer time evolution starting
from the hartree fock state <phi_hf| U(n dt) |phi_hf>, rather than explicity construction of Smn = <phi_hf| U(m dt)^ U(n dt) |phi_hf>
to reduce circuit depth.

There needs to be very clear deliniation between

A) Unitary quantum Krylov -> use full trotter blocks for each step, deeper circuits -> use MFE to determine elements of S
B) Stochastic unitary quantum Krylov -> use sampled blocks for each step, shallower circuits -> still use MFE to determine elements of S

NOTE: We will need intelegent handeling of the qiskit hardware permissions if jobs are run from the repo. Consult qiskit documentation and
outile a safe best practice for the user passing the permission keys we have from our IBM cloud account.  

8. Save the final S matrix as a npz file, along with any relevant metadata.

9. In another script similar in structure to that generated in 7. Build the unitary quantum krylov overap matrix direct linear algebra computation, using noisless qiskit simulator.
Then make sure that S-lin-alg is equivelant to S-MFE when full trotter blocks are used and no noise is introduced in the sumulator.

10. In a new script, read in the S matrix and perform the nececary arithmetic to generate the full generalized eigenvalue probelm (note the GEV will be 
of a smaller dimension than S). Use the same GEV thresholding procedure qforte uses to solve the GEV with better conditioning (consult qforte source
code for this). Perform the diagonalization, and save a summary of the origional molecule metadata (Ehf, and Efci for however many roots) and the energies predicted by the Unitary
quantum krylov procedure.
